Simulating Policy Scenarios using Aggregate Model

Applying the Aggregate Model to Real World Scenarios

While tackling the economic policy aspects of climate change, policymakers are required to take certain steps or announce policy measures that aim at reducing emissions from a particular economic activity. These policy measures can include announcing subsidies, tax breaks, investments in the activities that promote an environment-friendly outlook to the activity. On the other hand, the activities that are carbon-intensive and have a high environmental impact need to be reduced or completely abandoned as there is no more capacity for the planet to accommodate more emissions. The relevant policy measures in this case can be quantity limits, taxing the activity (or raising the tax), and in some cases even cutting off the activity.

In this vignette, I will simulate some of these policy scenarios using the Aggregate Model and will try to calculate the trajectory of emissions and the savings in emissions if these were to be exogenously pushed by the policymakers.

Policy Scenario 1 : Metal Production in Germany

Germany is the biggest producer of Metals in the European Union and is one of the top producers in the world markets too. Metal Production is also one of the most energy and emissions intensive economic activities.

In this scenario, I will simulate the effect of a policy proposal that aims at cutting metal production in Germany to 50% of its 2015 values by 2027, as part of its commitments to an international climate agreement.

Choice of Variables

In this specification, we have modelled Industrial Production, Industrial Emissions, and Emissions from Combustion. Since Metal Production is a significant contributor to the overall industrial production of Germany, a significant relationship could provide us with the effect our policy can have on overall Industrial Production. Similarly, for Industrial Emissions, the choice of using only Metal_Prod (and other non-prod indicators) is to observe the isolated effect of Metal_Prod on Emissions. HICP_Energy provides us a good proxy of energy prices which is believed to have a negative effect on emissions. The specification of model in this way allows us to choose a variety of combinations of variables and also check for whether the relationships we assume are statistically significant in this combination. Although, it is duly acknowledged that this way is vulnerable to Omitted Variable Bias much like other linear models, thus the outcome shouldn’t be necessarily interpreted causally.

Module Network

We can show the relationship between the different modules using the network() function.

network(model)

Each node represents a module and the different colours represent whether the variable is given by a definition/identity, whether it has been modelled as an endogenous variable depending on other models, and whether it is an exogenous variable input to the models.

An solid line arrow means that the variable has been retained during model selection, while a dashed arrow means that the variable has been found insignificant and thus dropped during model selection.

Forecasting (Simulating Policy Scenarios)

Now, this is where we can simulate policy scenarios based on the forecasted values of variables we are interested in. Firstly, we can use the forecast_model() function to forecast values of the different modules using an autoregressive (AR) function and save the results. Since, in this scenario assumption, we have 15 quarters to model we have set the parameter n.ahead=15.

f1 <- forecast_model(model, exog_fill_method = "AR", n.ahead = 15, plot.forecast = FALSE)
#> No exogenous values provided. Model will forecast the exogenous values with an AR4 process (incl. Q dummies, IIS and SIS w 't.pval = 0.001').
#> Alternative is exog_fill_method = 'last'.
plot(f1)

The following code is crucial for this scenario assumption. We have assumed that the cuts in production will progress over the quarters at a constant rate. To calculate this rate and therefore the indexed quantity of metal production, we have used a simple compounding formula which estimates a rate of 3.76% decline per quarter. In simple terms, the production of each subsequent quarter will be 3.76% less than the preceding quarter so that it finally reaches to 50% (of 2015) by the last quarter of 2027. It can be noted that this is only one way to reach this goal.

initial_value <- f1$exog_data_nowcast$Metal_Prod[1]
final_value <- 50 #Final Value is the desired policy implementation. That by 2027, the metal production should be 60% of that of the 2015 value.
periods <- 16  

rate <- (final_value / initial_value)^(1 / (periods - 1)) #In this case, this requires cutting the metal production by around 3.76% of the previous quarter

decreasing_values <- initial_value * rate^(0:(periods - 1))

Now we can apply these calculated values as exogenously provided values to forecast another model with these lower values of metal production to calculate savings in industrial emissions.

exog_data_high_ready <- f1$exog_data_nowcast %>%
  mutate(Metal_Prod = decreasing_values)

f1_high <- forecast_model(model, exog_predictions = exog_data_high_ready, plot.forecast = FALSE)

From this forecast, we can easily retain the data of two variables we are interested in, namely Industrial Emissions and Metal Production. The grepl_variables parameter in the plot() function matches the string combination to automatically capture the desired variable without the need of writing the exact variable name as it is saved. We can use this data to create a plot to visually present our forecast results of these variables under the exogenously provided metal production quantities.

f1_data <- plot(f1, exclude.exogenous = FALSE, grepl_variables = "Industry|Metal", return.data = TRUE)
f1_high_data <- plot(f1_high, exclude.exogenous = FALSE, grepl_variables = "Industry|Metal", return.data = TRUE)

plotting_scenario_data <- f1_data %>%

Finally calculating the savings in emissions. It can be noted a positive diff indicates the rise in emissions given the scenario values whereas a negative diff value suggests a fall in emissions under the scenario. In this case, we observe a very high diff of 106714 units between the AR forecasted values of metal production and those given exogenously.

f1_data %>%
  mutate(name = "AR Forecast") %>%
  bind_rows(f1_high_data %>%
              mutate(name = "Scenario Values")) %>%
  filter(na_item == "EmiCO2Industry" & type == "Endogenous Forecast") %>%
  summarise(emissions = sum(values, na.rm = TRUE), .by = name) %>%
  mutate(total = sum(emissions),
         diff = c(NA,diff(emissions)),
         rel_diff = diff/total)

Policy Scenario 2 : Energy Subsidies in Austria

In this scenario, a policy proposal that aims to subsidise energy prices in Austria is considered. This move has been a popular strategy for governments, during the times of high energy prices, to relieve households and businesses from high prices. Conversely, it has significant implications for GHG emissions.
